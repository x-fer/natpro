Meet in the middle

Autor: Petar Mihalj

Meet in the middle (MITM) skup je tehnika kojima prostor pretraživanje krenemo pretraživati s obe strane. Na primjer, traženje najkraćeg puta može istodobno krenuti od odredišta i cilja.

Problem 1: K-sum (MITM)​

Ideja korištenja MITM tehnike nad ovim problemom dolazi iz činjenice da bismo brojeve mogli podijeliti na 222 skupa, AAA i BBB, brojevima iz skupa AAA povećavati nulu, a brojevima iz skupa BBB smanjivati xxx.

Iako je ovo dobra formulacija rješenja, pretraga bi uključivala traženje svih stanja (zbrojeva) koji se mogu dobiti iz AAA - ovo stanja nazovimo SAS_ASA​. S druge strane, sva reduciranja xxx-a brojevima iz B rezultiraju stanjima koja nazivamo SBS_BSB​. Na kraju moramo pronaći broj ppp koji postoji i u SAS_ASA​ i u SBS_BSB​.

Na primjer, ako brojeve {3,5,7}\{3,5,7\}{3,5,7} zajedno sa ciljem 10 podijelimo u skupove A={3,5}A = \{3,5\}A={3,5} i B={7}B = \{7\}B={7}, rezultatna stanja su SA={0,3,5,8}S_A = \{0,3,5,8\}SA​={0,3,5,8} i SB={10,3}S_B = \{10,3\}SB​={10,3}. Budući da je broj 3 nalazimo u oba skupa stanja, odgovor na problem je "DA".

Traženje preklapanja možemo ostvariti sortiranjem jednog od skupova stanja (SAS_ASA​), a zatim iteriranjem po elementima SBS_BSB​, kojima tražimo para u SAS_ASA​ binarnom pretragom.

#include <bits/stdc++.h>
using namespace std;

int main(){
    int n,x;
    cin >> n;
    int a = n/2;
    int b = n-a;

    // pola u A
    vector<int> A(a);
    for (int i=0;i<a;i++){
        cin>>A[i];
    }
    
    // pola u B
    vector<int> B(b);
    for (int i=0;i<b;i++){
        cin>>B[i];
    }
    cin >> x;

    // izracunaj i zapamti S_A (from0)
    vector<int> from0;
    for(int i=0; i < (1<<a); i++){
        int sum = 0;
        for (int j=0;j<a;j++){
            if ((i & (1<<j)) != 0){
                sum += A[j];
            }
        }
        from0.push_back(sum);
    }

    // sortiraj S_A
    sort(from0.begin(), from0.end());

    // iteriraj po S_B
    for(int i=0; i < (1<<b); i++){
        int fromX = x;
        for (int j=0;j<b;j++){
            if ((i & (1<<j)) != 0){
                fromX -= B[j];
            }
        }
        // trazi para u S_A
        auto r = lower_bound(from0.begin(), from0.end(), fromX);
        if (r != from0.end() && *r == fromX){
            cout << "DA" << endl;
            return 0;
        }
    }

    cout << "NE" << endl;
    return 0;
}

Postoji i drugi pristup koji daje slično rješenje, a on se svodi na računanje oba skupa krećući od nule, ali tražeći x−tx-tx−t u SAS_ASA​, gdje je ttt trenutni element iz SBS_BSB​. Probajte i ovaj pristup!

Analiza složenosti​

Vremenska složenost ovog rješenja dobije se iz komponenata:

  1. Generiranje skupova O(2n/2⋅2)\mathcal{O}(2^{n/2} \cdot 2)O(2n/2⋅2)
  2. Pretraga O(2n/2⋅log⁡(2n/2))=O(2n/2⋅n2log⁡(2))\mathcal{O}(2^{n/2} \cdot \log(2^{n/2})) = \mathcal{O}(2^{n/2} \cdot \frac{n}{2} \log(2))O(2n/2⋅log(2n/2))=O(2n/2⋅2n​log(2))

Ukupna vremenska složenost je dakle O(2n/2⋅n)\mathcal{O}(2^{n/2} \cdot n)O(2n/2⋅n)

Ipak, rušenje eksponenta u vremenskoj složenosti na pola platili smo dizanjem prostorne složenosti na O(2n/2)\mathcal{O}(2^{n/2})O(2n/2). Primjetimo i da jedan skup stvarno treba zapamtiti (onaj po kojem tražimo), a drugi možemo generirati on-the-fly. Ovo je klasični primjer time-space tradeoff-a u algoritamskim tehnikama.
