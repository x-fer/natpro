Sparse table

Autor: Adrian Brajković

Uvod​

Sparse table je struktura podataka koja nad statičkim nizovima može obaviti min/max upite nad intervalima u O(1)O(1)O(1). Kako bismo to ostvarili, potrebno je nad nizom obaviti pretprocesiranje i izgraditi sparse table. Složenost pretprocesiranja je O(n∗log(n))O(n*log(n))O(n∗log(n)). Ovdje ćemo prikazati funkcionalnosti koristeći sparse table za min upite, a analogno se gradi i za max upite. Cilj nam je imati zapisano u matrici rješenja min upita za sve duljine koje su potencije broja 2. Dakle, imamo matricu sparseTable[i][j]sparseTable[i][j]sparseTable[i][j], gdje iii predstavlja indeks početka intervala, a jjj predstavlja eksponent pri potenciranju broja 2 što označava duljinu intervala.

Primjer:
sparseTable[3][0]sparseTable[3][0]sparseTable[3][0] - minimalni broj na intervalu koji počinje na indexu 3, a duljine je 20=12^{\smash{0}} = 120=1
sparseTable[2][1]sparseTable[2][1]sparseTable[2][1] - početak intervala je 2, a on je duljine 21=22^{\smash{1}} = 221=2
sparseTable[1][2]sparseTable[1][2]sparseTable[1][2] - početak intervala je 1, a on je duljine 22=42^{\smash{2}} = 422=4

Ako ovo prikažemo formulom:

sparseTable[i][j]=min(array[i],array[i+1],...,array[i+2j−1])sparseTable[i][j] = min(array[i], array[i + 1], ..., array[i + 2^{\smash{j}} - 1])sparseTable[i][j]=min(array[i],array[i+1],...,array[i+2j−1])

Pretprocesiranje​

Prvi korak je upisati rješenja za sve duljine 1 (kad je j=0j = 0j=0). U tom slučaju samo prepišemo vrijednosti iz početnog niza:

sparseTable[i][0]=array[i];sparseTable[i][0] = array[i];sparseTable[i][0]=array[i];

Imamo rješenja za intervale: [0,0],[1,1],[2,2],...[0, 0], [1, 1], [2, 2], ...[0,0],[1,1],[2,2],...

Sad računamo za intervale duljine 2 ([0,1],[1,2],[2,3],...)([0, 1], [1, 2], [2, 3], ...)([0,1],[1,2],[2,3],...).
Primijetimo da se svaki taj interval može izračunati pomoću intervala duljine 1 koje već imamo zapisane.
Minimalni broj na intervalu [0,1][0, 1][0,1] je manji od min[0,0]min[0, 0]min[0,0] i min[1,1]min[1, 1]min[1,1].
Nadalje, min[1,2]=min(min[1,1],min[2,2])min[1, 2] = min(min[1, 1], min[2, 2])min[1,2]=min(min[1,1],min[2,2]), i tako za svaki element niza (osim zadnjeg, jer iz njega ne može početi interval duljine 2).

Ovime smo završili pretprocesiranje i izgradili sparse table.
Formula:

sparseTable[i][j]=min(sparseTable[i][j−1],sparseTable[i+2j−1][j−1])sparseTable[i][j] = min(sparseTable[i][j-1], sparseTable[i + 2^{\smash{j-1}}][j-1])sparseTable[i][j]=min(sparseTable[i][j−1],sparseTable[i+2j−1][j−1])

Zašto baš i+2j−1i + 2^{\smash{j-1}}i+2j−1? Jer nam treba desna polovica intervala duljine 2j2^{\smash{j}}2j, dakle početnom indeksu pridodajemo pola duljine niza.

Programski kod​

vector < vector<int> > createSparseTable(vector <int> arr) {
	int N = arr.size();
	int K = log2(N); //2^K je najveca duljina intervala koja ne prelazi duljinu niza

	vector <int> row(K + 1);
	vector < vector<int> > sparseTable(N, row);

	for(int i = 0; i < N; i++) { // j = 0
		sparseTable[i][0] = arr[i]; 	  //intervali duljine 2^0 = 1
	}

	for(int j = 1; j <= K; ++j) {
		int len = pow(2, j); //duljina intervala

		for(int i = 0; i + len - 1 < N; ++i) { //uvjet je da interval [i, i + 2^j - 1] ne prelazi izvan granica niza
			sparseTable[i][j] = min(sparseTable[i][j - 1], sparseTable[i + len/2][j - 1]);
		}
	}


	return sparseTable;
}
Napomena

Često se umjesto pow(2,n)pow(2, n)pow(2,n) koristi logički posmak ulijevo (engl. bitshift) za potencije broja 2. (1<<n)(1 << n)(1<<n)

Upiti​

Kako sada odgovoriti na upit za najmanji broj nad nekim intervalom? Uzmemo najveću potenciju broja 2, a da ne prelazi duljinu intervala. Na primjer, tražimo rješenje za interval [2,7][2, 7][2,7]. Najveća potencija broja 2 u ovom slučaju je 4.
Vrijedi j=2j = 2j=2 i sad radimo upit za sparseTable[2][j]sparseTable[2][j]sparseTable[2][j] (početak intervala je 2, a duljina 2j=42^{\smash{j}} = 42j=4). Osim toga uzimamo upit i za desni dio intervala, točnije [4,7][4, 7][4,7], a to je sparseTable[7−2j+1][j]sparseTable[7 - 2^{\smash{j}} + 1][j]sparseTable[7−2j+1][j]. Uzmemo manju vrijednost i ona je rješenje.
Možemo vidjeti da ta dva upita uistinu pokrivaju cijeli interval.

Konačno, formula za dohvat najmanjeg broja na intervalu [L,R][L, R][L,R] (duljina intervala = R - L + 1):

min(sparseTable[L][j],sparseTable[R−2j+1][j])min(sparseTable[L][j], sparseTable[R - 2^{\smash{j}} + 1][j])min(sparseTable[L][j],sparseTable[R−2j+1][j]) j=floor(log2(R−L+1))j = floor(log2(R - L + 1))j=floor(log2(R−L+1))
